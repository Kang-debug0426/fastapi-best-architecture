## 十二、Bug 排查方法论

### 排查步骤框架

1. **复现问题**
   - 能否稳定复现？
   - 复现条件是什么？
   - 是否有最小复现案例？

2. **定位层次**
   - 客户端问题？网络问题？服务端问题？数据库问题？
   - 使用二分法缩小范围

3. **收集信息**
   - 查看日志（应用日志 / 系统日志 / 容器日志）
   - 检查监控指标（CPU / 内存 / 网络 / 磁盘）
   - 查看数据库慢查询日志

4. **假设验证**
   - 提出假设
   - 设计验证方案
   - 执行验证

5. **修复与验证**
   - 实施修复
   - 验证修复效果
   - 回归测试

6. **复盘总结**
   - 根本原因是什么？
   - 如何预防类似问题？
   - 是否需要添加监控告警？

### 常见问题排查清单

- [ ] 检查环境变量是否正确加载
- [ ] 检查服务是否正常启动（docker compose ps）
- [ ] 检查网络连通性（容器间能否互相访问）
- [ ] 检查数据库连接（连接池是否耗尽）
- [ ] 检查 Redis 连接（是否超时）
- [ ] 检查日志中的 ERROR 和 WARNING
- [ ] 检查请求/响应格式是否符合预期

| 方法 | 语义 | 幂等性 | 示例 |
|------|------|--------|------|

| PATCH | 部分更新 | 否 | PATCH /users/1 |
